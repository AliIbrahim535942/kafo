# تطبيق متجر إلكتروني متكامل (Full-Stack E-commerce)

هذا المشروع عبارة عن تطبيق متجر إلكتروني كامل يحتوي على:

- Backend باستخدام Node.js + Express + MongoDB Atlas
- Frontend باستخدام React + Bootstrap
- بنية واضحة وسهلة الفهم للمبتدئين (MVC في الخلفية + تنظيم صفحات/مكونات في الواجهة)

## 1) نظرة عامة على النظام

التطبيق يدعم سيناريو متجر إلكتروني حقيقي بشكل مبسط:

- تسجيل حساب وتسجيل الدخول
- استرجاع كلمة المرور (Forgot Password / Reset Password)
- عرض المنتجات مع البحث والتصفية حسب الفئة
- تفاصيل المنتج + التقييم مرة واحدة لكل مستخدم
- سلة مشتريات (إضافة/حذف/تعديل كمية + حساب الإجمالي)
- Checkout مع:
  - عنوان/موقع التسليم
  - دفع وهمي (Fake Payment)
  - 3 طرق توصيل
  - منطق توصيل مجاني إذا الطلب أكبر من 100 دولار
- إنشاء الطلبات والفواتير
- لوحة تحكم Admin لإدارة المنتجات والطلبات والفواتير والخصومات
- نظام نقاط وكوبونات:
  - كل 10$ = نقطة واحدة
  - كل 5 نقاط = كوبون 5$

## 2) هيكل المشروع

```bash
ecommerce-app/
  backend/
  frontend/
  README.md
```

## 3) كيف ترتبط الأجزاء معًا

- الواجهة الأمامية ترسل طلبات HTTP إلى API في الخلفية عبر Axios.
- الخلفية تتحقق من المدخلات (Joi) ثم تطبق منطق العمل (Services) ثم تتعامل مع MongoDB عبر Mongoose Models.
- JWT يستخدم لحماية المسارات الخاصة بالمستخدم أو الأدمن.
- الصور تُرفع محليًا داخل مجلد uploads وتُعرض بشكل static من السيرفر.

## 4) الميزات الرئيسية

- أدوار المستخدم:
  - admin
  - customer
- المنتجات:
  - title, description, price, image, barcode, rating, category, discount, bestseller
- الفئات:
  - 9 فئات محددة مسبقًا
- الطلبات والفواتير:
  - كل طلب ناجح ينشئ فاتورة
  - المستخدم يرى فواتيره
  - الأدمن يرى كل الفواتير

## 5) التشغيل السريع

### Backend

1. ادخل إلى مجلد backend
2. ثبّت الحزم: `npm install`
3. انسخ `.env.example` إلى `.env` وعدّل القيم
4. نفّذ seed: `npm run seed`
5. شغّل السيرفر: `npm run dev`

### Frontend

1. ادخل إلى مجلد frontend
2. ثبّت الحزم: `npm install`
3. انسخ `.env.example` إلى `.env`
4. شغّل الواجهة: `npm run dev`

## 6) بيانات Seed الافتراضية

- Admin (من خلال seed):
  - email: القيمة الموجودة في `ADMIN_EMAIL` داخل `.env`
  - password: القيمة الموجودة في `ADMIN_PASSWORD` داخل `.env`
- يتم إضافة مجموعة منتجات تجريبية تلقائيًا.

## 7) ملاحظات مهمة

- لا يوجد Docker حسب الطلب.
- الدفع وهمي فقط (لا يوجد بوابة دفع حقيقية).
- إذا كان رقم البطاقة ينتهي بـ `0` تعتبر العملية فاشلة (لأغراض التجربة).
- جميع الخطوات مكتوبة بطريقة مباشرة لتسهيل التعلم والتطوير لاحقًا.
